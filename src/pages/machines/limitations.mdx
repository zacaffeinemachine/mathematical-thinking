A machine has finitely many states. Push a string through it that
is longer than the state count, and by pigeonhole some state must
be visited twice. The block of input read between those two visits
is a **loop**: structurally, the machine cannot tell whether it
went around that loop *zero, one, or twenty* times.

## The Loop Lemma

Let *M* be a machine with *k* states, and let *s* be a string that
*M* accepts with |*s*| ≥ *k*. Then *s* admits a decomposition

<div style={{textAlign:'center', fontFamily:'ui-monospace, monospace', margin:'8px 0', fontSize:16}}>
s = u v w
</div>

with *v* non-empty, such that for every integer *q* ≥ 0, the string
*u v<sup>q</sup> w* is also accepted.

</div>

The picture is exactly this:

Reading *u* drives the machine from the start state q₀ to some
state qᵢ. Reading *v* takes qᵢ back to qᵢ. Reading *w* finishes
the trip to the accepting state qₙ. Pumping *v* — replaying it
*q* times instead of once — never changes where we end up.

## Three impossibilities

Pumping is fatal for any language whose accepted lengths refuse to
sit on a long enough arithmetic progression. Three natural
languages fall to it.

**0ⁿ1ⁿ.** Strings with the same number of `0`s as `1`s, all the
zeros first. Take *s* = `0`<sup>k</sup>`1`<sup>k</sup>; the loop
*v* must lie inside the leading run of zeros, so pumping it adds
zeros without adding ones. The accepted string `0`<sup>k+m</sup>`1`<sup>k</sup>
breaks the count.

**A square number of `1`s.** Take *s* = `1`<sup>m²</sup> with
*m* > *k*. The lemma promises that m², m²+p, m²+2p, … are all
accepted, with 1 ≤ *p* ≤ *k*. But m² + *p* lies strictly between
m² and (m+1)² = m² + 2m + 1, so it is not a square.

**A prime number of `1`s.** Take *s* = `1`<sup>P</sup> with *P*
prime and *P* > *k*. The lemma promises P + qp prime for every q.
Set q = P: then P + Pp = P(1+p), composite. Both factors are at
least 2.

## The line

The first half of this chapter showed what machines *can* do —
recognise any property expressible with bounded memory, and
combine such properties freely via the product construction. The
Loop Lemma names the other side of the line. A property that
requires *unbounded* memory — matching counts, recognising
squares, recognising primes — falls to a single weapon: take a
long-enough accepted string, find the loop, pump.

This boundary is structural, not a failure of cleverness. Crossing
it requires a different kind of machine — a story for later
chapters.
