This looks like just another flavour of the product construction.
But it has a striking consequence.

## When do two machines recognise the same language?

Suppose two engineers each build a machine — call them *M* and *N*
— and they want to know whether the two machines accept exactly
the same strings.

Tested by hand, the question seems hopeless: there are infinitely
many strings, and verifying agreement on the first thousand says
nothing about the thousand-and-first.

**The XOR trick.** Build the XOR of *M* and *N*. By construction
this machine accepts a string if and only if *M* and *N* disagree
on it. So:

> *M* and *N* recognise the same language &nbsp;⟺&nbsp; the XOR machine accepts **no string at all.**

And *that* is a finite question. It does not refer to strings — it
asks whether, starting from the start state, there is any path of
arrows that reaches an accepting state. With a finite graph, you
can answer it by sweeping outward from the start until no new
states show up.

We have replaced an infinite collection of yes/no questions
("agree on each of these strings?") with a single finite check
("walk the wires of one diagram"). That is the kind of trade
mathematicians find irresistible.
