We have been building machines for a property. Now we flip the
game: here is a machine. **What does it accept?**

The machine will not tell you what it is thinking. The two moves
that work are:

1. **Experiment.** Feed in short strings, in order: empty,
   <code>0</code>, <code>1</code>, <code>00</code>, <code>01</code>, … 
   A pattern usually announces itself within ten or twelve attempts.
2. **Name the states.** For each circle, ask *what must be true of
   the input so far for the machine to be sitting here?* When the
   names are right, the arrows click into place.

## Mystery machine

<FSMRunner
  client:visible
  machine={mysteryMachine}
  defaultInput="1011"
  examples={["", "0", "1", "00", "01", "10", "11", "101", "110", "1011"]}
/>

<details style={{margin:'12px 0', padding:'12px 16px', border:'1px solid var(--rule)', borderRadius:6}}>
<summary style={{cursor:'pointer', fontWeight:500}}>Reveal — what this machine accepts</summary>

It accepts every string that contains <code>11</code> as a substring.

- **<code>s0</code>** — the input so far does not end in
  <code>1</code>. (The empty string lives here, and so do
  <code>0</code>, <code>100</code>, <code>0010</code>, …)
- **<code>s1</code>** — the input ends in exactly one
  <code>1</code>.
- **<code>s2</code>** — we have already seen <code>11</code>
  somewhere. Both arrows from <code>s2</code> loop back: once a
  <code>11</code> has appeared, the verdict is locked in.

</details>

## A subtler language

A second machine. It runs *two parity counters at once* — but the
state names give that away if you stare long enough. Try the empty
string, then <code>0</code>, <code>1</code>, <code>00</code>,
<code>11</code>, <code>0011</code>, …

<FSMRunner
  client:visible
  machine={evenZerosEvenOnes}
  defaultInput="0011"
  examples={["", "0", "1", "00", "01", "11", "0011", "0110", "001", "010"]}
/>

<details style={{margin:'12px 0', padding:'12px 16px', border:'1px solid var(--rule)', borderRadius:6}}>
<summary style={{cursor:'pointer', fontWeight:500}}>Reveal — what this machine accepts</summary>

Every string with an **even number of <code>0</code>s** *and* an
**even number of <code>1</code>s**. The first letter of each state
name tracks the parity of <code>0</code>s (E for even, O for
odd); the second letter tracks the parity of <code>1</code>s.
The only accepting state is <code>EE</code>.

This is a tiny preview of the **product construction** — running
two parity machines on the same tape — that we'll meet two pages
on.

</details>
